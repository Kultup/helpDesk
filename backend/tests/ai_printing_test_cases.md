# Тестові кейси для валідації AI-логіки друку

## Проблемні сценарії (повинні створювати тікет)

1. **"не можу друкувати в ворд"** → PROBLEM, створювати тікет
2. **"ворд не друкує"** → PROBLEM, створювати тікет
3. **"помилка друку з ворда"** → PROBLEM, створювати тікет
4. **"принтер не друкує взагалі"** → PROBLEM, створювати тікет
5. **"проблема з друком документів"** → PROBLEM, створювати тікет
6. **"не виходить надрукувати таблицю"** → PROBLEM, створювати тікет
7. **"друк завис"** → PROBLEM, створювати тікет

## HOW-TO сценарії (повинні давати інструкції)

1. **"як роздрукувати документ"** → HOW-TO, інструкція Ctrl+P
2. **"де знаходиться друк в ворді"** → HOW-TO, інструкція Файл→Друк
3. **"як надрукувати з word"** → HOW-TO, інструкція 3 кроки
4. **"shortcut для друку"** → HOW-TO, інструкція Ctrl+P
5. **"як знайти панель друку"** → HOW-TO, інструкція розташування

## Крайові випадки

1. **"не можу знайти друк в ворді"** → PROBLEM (не можу + друк)
2. **"як роздрукувати але щось не працює"** → PROBLEM (не працює)
3. **"покажіть як друкувати будь ласка"** → HOW-TO (немає негативних маркерів)
4. **"друк не працює як в інструкції"** → PROBLEM (не працює)

## Правило валідації

Якщо текст містить будь-який з негативних маркерів:

- "не можу", "не працює", "проблема", "помилка", "завис", "не вийшло", "не вдається"

→ ЗАВЖДИ PROBLEM, незалежно від наявності слів "як", "ворд", "документ"
